問47


〔問１〕
●CSMA/CD方式(Carrier Sense Multiple Access with Collision Detection)
：複数のデバイスが同じ通信回線を共有するためのアクセス制御方式
通信の効率を最大化するために衝突を検出し、再送信を行うことで安定した通信を確保
●TDMA方式(Time Division Multiple Access)
：時間を分割して通信回線を複数のユーザーが共有する方式


〔問３〕
IPアドレス・クラス



〔問４〕
●PVC(Permanent Virtual Circuit)
：仮想回線の一種で、特定の通信経路を仮想的に設定し、常に接続されている状態を維持する方式
通常は企業内ネットワークやプロバイダと接続する際に使用



〔問６〕
●SNMP(Simple Network Management Protocol)
：ネットワーク機器（ルータ、スイッチ、サーバ、プリンタなど）の監視と管理を行うためのアプリケーション層に属する通信プロトコル
ネットワーク機器の監視や設定変更、性能の監視、アラート通知
●ICMP(Internet Control Message Protocol)
：インターネット層のプロトコルで、ネットワーク通信のエラー報告や診断に使用されるインターネット層に属するプロトコル
ネットワークの障害検出や接続テスト



〔問９〕
●VRRP(Virtual Router Redundancy Protocol)
：ルータの冗長化技術の一つで、複数のルータをグループ化して、1台のルータが故障してもネットワークの通信が途切れないようにするプロトコル


〔問10〕
●RIP(Routing Information Protocol)
：距離ベクトル型のルーティングプロトコルで、ネットワーク内でデータパケットを転送する最適な経路を決定するために使用
IPネットワークでルータ同士がルーティング情報を交換するために利用




〔問12〕
●データグラム
：通信ネットワークにおける最小単位のデータで、送信元から宛先へ送られる情報の単位
UDPで使用される


〔問15〕
●ネットワークインタフェースカード(NIC：Network Interface Card)
：コンピュータをネットワークに接続するための装置で、LANケーブルやWi-Fiを通じて、データの送受信を行う
●ターミネータ
：主に古いネットワーク規格（例：同軸ケーブル）で使われていた、信号の反射を防ぐ終端装置
●ハブ
：複数のコンピュータを接続する集線装置
●トランシーバ
：送信（Transmitter）と受信（Receiver）を一体化した通信装置で、LANケーブルなどで送受信信号を変換する役割を持つ
●モデム
：デジタル信号とアナログ信号を相互変換する装置で、主に電話回線を使ってインターネットに接続する際に使われ、今でもADSLなどの環境で見られる
●MAU(Media Access Unit または Multi-station Access Unit)
：ネットワークの中継装置や接続装置の一種
・Media Access Unit
：イーサネットで使われた物理的な接続装置
・Multi-station Access Unit
：トークンリングで使われたスター型の中継装置


〔問17〕
●FTTH(Fiber To The Home）)
：光ファイバーを家庭まで引くインターネット接続方式
●ONU(Optical Network Unit)
：光回線の終端装置で、光信号を電気信号に変換する役割


〔問20〕
●PCM(Pulse Code Modulation：パルス符号変調)
：音声やアナログ信号をデジタル信号に変換する方式のこと


〔問22〕
●ダイヤルアップルータ
：電話回線を使ってインターネット接続するためのルータ
●ブロードバンドルータ
：常時接続の高速インターネット回線（ADSL、光回線、CATVなど）に対応したルータ



〔問27〕
●IMAP4(Internet Message Access Protocol version 4)
：電子メールを受信・管理するためのプロトコル（通信規約）
メールを サーバ上に保存したまま操作 できるのが特徴
例：Gmailなどのクラウド型メールサービス
●RSVP(Resource Reservation Protocol)
：ネットワークで通信経路上の帯域（リソース）を予約するためのプロトコル
動画や音声など、リアルタイム性が重要な通信で使用されることを想定



〔問33〕
●NNTP(Network News Transfer Protocol)
：ネットニュース（電子掲示板）の投稿・取得に使う通信プロトコル
●NTP(Network Time Protocol)
：コンピュータの時刻を正確に保つためのプロトコル
●SNMP(Simple Network Management Protocol)
：ネットワーク機器（ルータやスイッチなど）を監視・管理するためのプロトコル


〔問35〕
●SAML(Security Assertion Markup Language)
：異なるドメイン間でユーザー認証情報を安全にやり取りするためのXMLベースの規格



〔問38〕
●CHAP(Challenge Handshake Authentication Protocol)
：接続時に、サーバーがクライアントにランダムなチャレンジ（挑戦）を送り、クライアントは自分のパスワードとそのチャレンジを元に計算した応答をサーバーに返す
●ICMP(Internet Control Message Protocol)
：IPネットワークでのエラーメッセージや診断メッセージの送受信に使用される通信プロトコル
 ICMPメッセージはエラーレポートや診断情報を提供するため、接続確認やルーティングの問題を特定する際に重要
●SNMP(Simple Network Management Protocol)
：ネットワーク機器の監視や管理を行うためのプロトコル
ルーター、スイッチ、サーバーなどのネットワーク機器を監視し、パフォーマンスや状態を管理者が確認
管理対象のデバイス（エージェント）に対して、監視、警告、設定変更を行えるため、ネットワーク管理に不可欠な技術



〔問39〕
●iBurst
：モバイル通信技術の一つで、特にデータ通信に優れたパフォーマンスを提供
固定のアクセスポイントから直接ユーザー端末にデータを送信する無線通信方式
●WiMAX(Worldwide Interoperability for Microwave Access)
：無線通信規格の一つで、広範囲のエリアにわたって高速インターネット接続を提供することを目的
固定とモバイルの両方のアプリケーションに適しており、高速インターネット接続を提供するため、特に都市部や地方でのブロードバンドアクセスに利用
●W-CDMA(Wideband Code Division Multiple Access))
：3G（第3世代）携帯電話システムで使用される通信方式

●次世代PHS(Next-generation Personal Handy-phone System)
：従来のPHS（パーソナル・ハンディ・フォン・システム）を進化させた通信技術で、音声通話だけでなく、高速データ通信も可能にした次世代のモバイル通信システム



〔問40〕
●IMAP4
：メールをサーバーで管理し、複数デバイスで同期が可能
●MIME
：メールの本文や添付ファイルに複数のメディア形式を対応させる規格
●POP3
：メールをサーバーからダウンロードし、ローカルに保存するプロトコル
●SMTP
：メールを送信するためのプロトコル


〔問44〕
●代替HTTPポート
：代わりに使われるポート番号のこと


〔問45〕
●1000BASE-T
：ギガビット・イーサネットの通信規格の1つ
●UTPケーブル(Unshielded Twisted Pair)
：ノイズにやや弱いが安価・LAN用で、オフィスや家庭のネットワークで利用
●マルチモード光ケーブル
：近距離・安価な光通信で、建物内ネットワークなどで利用
●シングルモード光ケーブル
：長距離・高性能光通信で、通信会社の基幹ネットワークで利用
●同期ケーブル
：データとクロックを同期で、専用通信機器（ATMなど）で利用


〔問48〕
●WEP(Wired Equivalent Privacy)
：無線LAN（Wi-Fi）の初期の暗号化方式
●WPA(Wi-Fi Protected Access)
：WEPの弱点を改善したより安全な暗号化方式



〔問49〕
●IPスプーフィング
：他人のIPアドレスを偽って通信する行為


〔問50〕
●周波数チャネル番号
：無線通信（Wi-FiやBluetoothなど）で使われる周波数の割り当て番号のこと
●ハンドオーバー
：移動中の通信を途切れさせずに、別の基地局やアクセスポイントへ切り替える仕組み

＜モデル駆動開発（MDD: Model-Driven Development）　概念ツリー＞

モデル駆動開発（MDD）
├── モデル
│   ├── 概念モデル（CIM: Computation Independent Model）
│   ├── 設計モデル（PIM: Platform Independent Model）
│   ├── 実装モデル（PSM: Platform Specific Model）
│   └── UML（Unified Modeling Language）
│
├── 変換
│   ├── モデル変換（M2M: Model-to-Model）
│   ├── モデルからコード生成（M2T: Model-to-Text）
│   └── 逆変換（T2M: Text-to-Model、リバースエンジニアリング）
│
├── ツール
│   ├── モデリングツール（例: Enterprise Architect、MagicDraw）
│   ├── モデル変換エンジン（例: ATL、QVT）
│   ├── コード生成ツール（例: Acceleo）
│   ├── フレームワーク（例: Eclipse EMF、Papyrus）
│   └── ドメイン固有言語（DSL: Domain-Specific Language）
│
├── 関連手法
│   ├── ラウンドトリップエンジニアリング（設計↔実装の双方向変換）
│   ├── メタモデリング（モデルのモデルを定義）
│   ├── ドメイン駆動設計（DDD: Domain-Driven Design）
│   └── ソフトウェアファクトリ
│
└── 標準・規格
    ├── OMG（Object Management Group）
    │   ├── MDA（Model-Driven Architecture）
    │   ├── MOF（Meta-Object Facility）
    │   ├── QVT（Query/View/Transformation）
    │   └── XMI（XML Metadata Interchange）
    └── ISO/IEC 19510（BPMN: Business Process Model and Notation）

│       │   │   ├── LRU（Least Recently Used）
│       │   │   └── LFU（Least Frequently Used）
│       └── セグメンテーション
│           ├── セグメントテーブル
│           ├── セグメントフォールト
│           └── セグメントベースのメモリ管理
├── メモリ管理
│   ├── 仮想メモリと実メモリの関係
│   ├── スワッピング
│   ├── デマンドページング
│   ├── メモリのフラグメンテーション
│   │   ├── 内部フラグメンテーション
│   │   └── 外部フラグメンテーション
│   ├── メモリ圧縮
│   ├── メモリキャッシュ
│   └── メモリプール
├── ストレージとの関係
